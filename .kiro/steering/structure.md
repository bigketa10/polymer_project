# Project Structure

The entire application lives under the `app/` directory.

```
app/
├── app/                        # Next.js App Router
│   ├── layout.tsx              # Root layout — wraps ClerkProvider + ConvexClientProvider
│   ├── page.tsx                # Entry point — renders PolymerChemistryApp (auth'd) or Homepage
│   └── teacher/
│       └── page.tsx            # Teacher dashboard route
│
├── components/
│   ├── ui/                     # shadcn/ui primitives (button, card, alert, progress)
│   ├── ConvexClientProvider.tsx # Convex + Clerk provider setup
│   ├── Homepage.tsx            # Unauthenticated landing page
│   ├── PolymerChemistryApp.tsx # Main student app shell
│   ├── Teacher.tsx             # Full teacher dashboard (TeacherDashboard component)
│   ├── StudentLeaderboard.tsx  # Leaderboard UI
│   ├── DragDropStudentQuestion.tsx # Drag & drop question type
│   ├── ExplainerText.tsx       # Post-answer explanation display
│   └── challenges/
│       └── CalculationCard.tsx # Calculation challenge component
│
├── convex/                     # Convex backend (all server-side logic)
│   ├── schema.ts               # Database schema — source of truth for all tables
│   ├── auth.config.js          # Clerk provider config for Convex auth
│   ├── auth.ts                 # Auth helpers
│   ├── lessons.ts              # Lesson CRUD + question management
│   ├── modules.ts              # Module CRUD + reordering
│   ├── lessonAttempts.ts       # Per-lesson attempt tracking
│   ├── userProgress.ts         # XP, streaks, leaderboard
│   ├── teachers.ts             # Class stats, student management
│   ├── glossary.ts             # Glossary CRUD
│   ├── uploads.ts              # Convex file storage helpers
│   ├── messages.ts             # Messages table
│   ├── users.ts                # User helpers
│   └── _generated/             # Auto-generated by Convex CLI — do not edit
│
├── lib/
│   └── utils.ts                # cn() helper (clsx + tailwind-merge)
│
├── public/
│   └── sounds/                 # correct.mp3, incorrect.mp3
│
├── .env.local                  # Local env vars (not committed)
└── package.json
```

## Key Conventions

- All pages use the Next.js App Router (`app/` directory). No `pages/` directory.
- Client components are marked with `"use client"` at the top. Convex hooks require this.
- Convex functions are imported via `api` from `../convex/_generated/api` — never call Convex HTTP directly.
- `identity.subject` (from Clerk via Convex) is used as the user ID everywhere — not Clerk's `user.id`.
- New UI components go in `components/ui/` if they are generic primitives, or directly in `components/` if feature-specific.
- Schema changes in `convex/schema.ts` require running `npx convex dev` to regenerate `_generated/`.
